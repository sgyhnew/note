fightgame，尚未命名，下文简称为FG
# 1.0
基于石头剪刀布的运作模式，演变为兵器和招式对决，用户通过输入来进行"猜拳"。

FG通过react模块来提取输入的关键字来输出对应招式。在目前版本只存在拳、剑、刀。其克制关系为拳可剑、剑可刀、刀可拳。

# 1.1
增加除招式外额外选项，现已有'帮助'和'逃跑'两个额外选项，出现关键词就会触发相应对话对玩家进行提示。并内置更多进阶招式，通过隐性的内置积分触发。

# 1.2
用户输入提取关键字不是很美观，用户大量输入却只能选择其中的一个关键词，因此决定放弃自主输入，而是通过声明menu提供选择菜单。并更改react函数的使用逻辑。同时内置多个函数

# 1.3
美化菜单页面，使用内部类Menu来管理菜单页面，现在菜单页面分为主菜单和攻击菜单，并增加更多内置函数

# 1.4
增加新内部类Attribute并内置多个函数，增加对血量和伤害的描述，游戏可分出胜出而不必每次都按逃跑

# 1.5
## 1.5.1
实装defense防御，并增加伤害和受击反馈，修复多个bug
## 1.5.2
将积分作为Attribute中的一个新属性能量而非Game大类
## 1.5.3
