4.fiddler
	概念
		'小提琴'，
	作用
		1.使用fiddler抓取数据来判断前后端bug
		2.没有开放文档时，可进行接口信息抓取
		3.模拟弱网测试
		4.app闪退时，可由fiddler抓包
		5.模拟断点操作
	工作原理
		在服务端和客户端生成一个代理服务器，端口号默认8888
	界面
		工具面板
			replay 重新执行抓取请求
			`❌` removeall 清空会话中所有内容 
			go 释放断点/拦截的请求
			stream 流模式，请求发送时存在顺序，抓取时按顺序展示请求信息
			decode 反编码/解码
			keep ：All sessions 保持所有会话
			find 查找，过滤请求得到目标会话
		会话面板
		result 状态码
			200：代表请求成功，但不代表结果数据正确
			302：请求重定向
			404：请求资源找不到
			500：服务器异常
			502：网关异常
			504：处理超时
		protocol 协议
		hosts 域名、服务器名字
		url 资源路径
		监控面板
